# 제안서 작성 프레임워크

> 인테리어/공간디자인 프로젝트 제안서 체계.
> 참고: harness-100 #86 proposal-writer 기반, 공간디자인 특화.

## 1. 제안서 작성 프로세스

```
고객 분석 → 컨셉 설계 → 공간 솔루션 → 가격 전략 → 차별화 → 통합 편집
```

## 2. 고객 분석 프레임워크

### 니즈 파악 질문 체계
| 단계 | 질문 유형 | 예시 |
|------|----------|------|
| **현재** | 현재 공간의 불만 | "지금 공간에서 가장 불편한 점은?" |
| **목표** | 원하는 변화 | "이 공간에서 어떤 경험을 만들고 싶으세요?" |
| **제약** | 예산/일정/규제 | "예산 범위와 희망 완공일은?" |
| **감성** | 분위기/가치관 | "어떤 느낌의 공간을 좋아하시나요?" |
| **기능** | 실용적 요구 | "일일 예상 고객 수, 좌석 수는?" |

### 의사결정 구조 파악
| 확인 사항 | 내용 |
|----------|------|
| 최종 결정자 | 대표? 담당자? 위원회? |
| 의사결정 기준 | 가격? 디자인? 일정? |
| 경쟁 상황 | 다른 업체 견적 받았는지 |
| 일정 압박 | 오픈일 확정 여부 |

## 3. 컨셉 설계 구조

### 스토리 기반 컨셉 양식
```
[공간 스토리]
══════════════════════════════
■ 브랜드 키워드: (3개)
■ 공간의 한 문장: "이 공간은 ___한 곳이다"
■ 경험 시나리오: 고객이 문을 열고 들어와서 나가기까지
■ 핵심 감성: (2~3개 형용사)
■ 디자인 원칙: (3개)
```

### 컨셉 설득 구조 (AIDA)
| 단계 | 목적 | 제안서 반영 |
|------|------|-----------|
| **A**ttention | 주목 끌기 | 강렬한 비주얼/한 문장 |
| **I**nterest | 관심 유발 | 문제 정의 + 인사이트 |
| **D**esire | 욕구 자극 | 완성 이미지, 경쟁 우위 |
| **A**ction | 행동 유도 | 다음 단계, 일정, 견적 |

## 4. 제안서 표준 구성

```
[프로젝트 제안서]
══════════════════════════════
1. 표지 (프로젝트명, 일자, 당사 로고)
2. 인사말
3. 프로젝트 이해 (고객 니즈 분석)
4. 컨셉 제안 (스토리 + 무드보드)
5. 공간 계획 (평면도, 동선, 조닝)
6. 디자인 제안 (마감재, 가구, 조명, 컬러)
7. 시공 계획 (일정, 공정표)
8. 견적 (공종별, 총액)
9. 당사 소개 (포트폴리오, 강점)
10. 별첨 (참고 이미지, 자재 샘플)
```

## 5. 차별화 전략 (Win Theme)

### USP 도출 방법
| 질문 | 당사 답변 |
|------|----------|
| 우리만 할 수 있는 것? | (기술/경험/네트워크) |
| 고객이 가장 중요하게 생각하는 것? | (니즈 분석 결과) |
| 경쟁사 대비 우위? | (가격/품질/디자인/일정) |

### Win Theme 문장 구조
```
"[고객 니즈]를 [당사만의 방법]으로 해결하여 [구체적 가치]를 실현합니다."
```

## 6. 산출물 양식

```
[제안서 작성 보고서]
══════════════════════════════
1. 고객 분석 요약
2. 컨셉 제안 (스토리 + 키워드)
3. 공간 계획 개요
4. 견적 요약
5. 차별화 포인트
6. 김전무 보고용 핵심 요약
```

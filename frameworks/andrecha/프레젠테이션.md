# 프레젠테이션 프레임워크

> 디자인 제안 프레젠테이션 구성 및 스토리텔링 체계.
> 참고: harness-100 #13 presentation-designer 기반.

## 1. 프레젠테이션 구조 (스토리 아크)

```
도입 (WHY) → 문제 정의 → 인사이트 → 솔루션 → 증거 → 행동 유도
```

### 디자인 제안 발표 표준 구성
| 섹션 | 슬라이드 수 | 내용 |
|------|:---------:|------|
| 1. 오프닝 | 1~2 | 프로젝트명, 한 줄 컨셉 |
| 2. 고객 이해 | 2~3 | 니즈 분석, 현재 문제 |
| 3. 컨셉 제안 | 3~5 | 스토리, 무드보드, 키워드 |
| 4. 공간 계획 | 3~5 | 평면도, 동선, 조닝 |
| 5. 디자인 디테일 | 5~8 | 마감재, 가구, 조명, 컬러 |
| 6. 시공 계획 | 2~3 | 일정, 공정표 |
| 7. 예산 | 1~2 | 공종별 견적, 총액 |
| 8. 클로징 | 1~2 | 다음 단계, 연락처 |
| **합계** | **18~30장** | |

## 2. 슬라이드 디자인 원칙

### 1장 1메시지 원칙
- 한 슬라이드에 하나의 핵심 메시지만
- 텍스트는 **최대 5줄, 25단어**
- 나머지는 이미지/도면으로

### 비주얼 계층
| 요소 | 용도 | 크기 |
|------|------|------|
| 제목 | 섹션 구분 | 가장 큼 |
| 키 비주얼 | 핵심 이미지/도면 | 슬라이드의 60~70% |
| 설명 텍스트 | 보조 설명 | 최소한으로 |
| 캡션 | 이미지 출처/주석 | 가장 작게 |

## 3. 스토리텔링 기법 (PREP)

| 단계 | 용도 | 예시 |
|------|------|------|
| **P**oint | 결론 먼저 | "이 공간의 컨셉은 '도시 속 숲'입니다" |
| **R**eason | 이유 | "타깃 고객은 30대 직장인, 자연에 대한 갈증" |
| **E**xample | 사례/증거 | "유사 컨셉의 A카페는 일매출 40% 증가" |
| **P**oint | 결론 반복 | "도시 속 숲 컨셉으로 차별화하겠습니다" |

## 4. 발표 노트 양식

```
[슬라이드 N] 제목
━━━━━━━━━━━━━━━
■ 핵심 메시지: (한 문장)
■ 발표 멘트: (30초 분량)
■ 예상 질문: (1~2개)
■ 답변 준비: (각 20초)
■ 전환 멘트: (다음 슬라이드로 넘기며)
```

## 5. 산출물 양식

```
[프레젠테이션 구성 보고서]
══════════════════════════════
1. 발표 목적 및 청중 분석
2. 스토리 아크
3. 슬라이드 구성표 (섹션별)
4. 핵심 슬라이드 내용 (텍스트)
5. 발표 노트
6. 김전무 보고용 핵심 요약
```

# 회의 전략 프레임워크

> 회의 안건 설계, 의사결정, 팔로업 체계.
> 참고: harness-100 #84 meeting-strategist 기반.

## 1. 회의 설계 프로세스

```
목적 정의 → 안건 설계 → 배경자료 준비 → 진행 → 결정사항 정리 → 팔로업
```

## 2. 회의 유형별 설계

| 유형 | 목적 | 시간 | 산출물 |
|------|------|:----:|--------|
| 의사결정 회의 | 안건 결정 | 30~60분 | 결정사항 + 실행계획 |
| 보고 회의 | 현황 공유 | 15~30분 | 이슈 + 지시사항 |
| 브레인스토밍 | 아이디어 도출 | 30~60분 | 아이디어 리스트 |
| 발주처 미팅 | 협의/협상 | 60~90분 | 합의사항 + 후속조치 |

## 3. 안건 설계 양식

```
[회의 안건]
══════════════════════════════
■ 회의명:
■ 일시:
■ 참석자:
■ 목적: (1문장)

■ 안건 1: [제목] (결정/보고/논의)
  - 배경:
  - 선택지: A안 / B안
  - 권고안: (사전 분석 결과)
  - 예상 소요: 분

■ 안건 2: [제목]
  ...

■ 필요 사전 자료:
■ 예상 총 소요시간:
```

## 4. 의사결정 프레임워크 (회의 중)

### 신속 결정 (RAPID)
| 역할 | 의미 | 담당 |
|------|------|------|
| **R**ecommend | 권고안 제시 | 담당자 |
| **A**gree | 동의 필요 | 관련 팀 |
| **P**erform | 실행 | 실행 담당 |
| **I**nput | 의견 제공 | 참고인 |
| **D**ecide | 최종 결정 | 대표님 |

## 5. 발주처 미팅 전략

### 미팅 전 준비
| 항목 | 내용 |
|------|------|
| 상대 의사결정 구조 | 누가 결정권자? |
| 상대 관심사 | 가격? 일정? 품질? |
| 우리 목표 | 이번 미팅에서 반드시 얻을 것 |
| 양보 가능 항목 | 양보 카드 준비 |
| 양보 불가 항목 | 사수 라인 |

### 미팅 후 정리
```
[미팅 결과 보고]
══════════════════════════════
1. 합의 사항 (명확히)
2. 보류 사항 (이유 + 다음 단계)
3. 상대방 반응/분위기
4. 후속 조치 (담당 + 기한)
5. 대표님 보고용 핵심 요약
```

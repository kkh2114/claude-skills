# CARL 형식 참조 v1

## 정의

**CARL** = Context → Action → Result → Learning

반복되는 판단에서 교훈을 구조화하여 축적하는 형식.
70-playbook/의 patterns/, anti-patterns/, insights/에서 사용.

## 왜 CARL인가

비구조화된 "메모"는:
- 나중에 찾기 어렵다
- 맥락이 빠져서 의미를 잃는다
- 에이전트가 참조할 수 없다

CARL은 4단계로 구조화하여:
- 검색 가능 (Context 키워드로)
- 재현 가능 (Action이 구체적)
- 학습 가능 (Learning이 명시적)
- 에이전트가 다음 분석에서 자동 참조 가능

## 형식

### Anti-Pattern (실패 교훈)

```markdown
# [교훈 제목] — YYYY-MM-DD

## Context (상황)
[어떤 시장 상황, 조건, 배경이었는가]

## Action (행동)
[어떤 판단을 했는가, 무엇을 근거로 했는가]

## Result (결과)
[실제로 어떤 결과가 나왔는가]

## Learning (교훈)
[다음에 같은 상황이 오면 어떻게 해야 하는가]

## 적용 조건
[이 교훈이 적용되는 상황 키워드: 예) 전세가율, 역전세, 금리상승]
```

### Pattern (성공 패턴)

```markdown
# [패턴 제목] — YYYY-MM-DD

## Context
[어떤 상황에서 이 패턴이 효과적이었는가]

## Action
[어떤 접근법을 사용했는가]

## Result
[어떤 긍정적 결과를 얻었는가]

## Learning
[이 패턴을 반복 적용할 때 유의할 점]

## 적용 조건
[이 패턴이 적용되는 상황 키워드]
```

### Insight (관찰 메모)

Insight는 CARL 전체를 사용하지 않아도 된다.
간단한 관찰은 자유 형식으로 기록:

```markdown
# [인사이트 제목] — YYYY-MM-DD

[관찰 내용]

**출처**: [데이터 출처]
**관련 키워드**: [검색용 키워드]
```

## 실제 예시

### Anti-Pattern 예시

```markdown
# 전세가율만 보고 진입한 역전세 — 2026-01-15

## Context
전세가율 75%, 금리 상승기, 입주물량 예정.
"갭이 적으니 안전하다"고 판단.

## Action
전세가율만 보고 매수 결정.
입주물량 확인 안 함. 금리 방향 체크 안 함.

## Result
입주 물량 증가 + 금리 상승으로 역전세.
추가 자금 투입 필요.

## Learning
전세가율 단일 지표만으로 판단하면 안 된다.
반드시 향후 입주물량 + 금리 방향을 함께 봐야 한다.

## 적용 조건
전세가율, 갭투자, 역전세, 입주물량
```

## 에이전트 활용 방식

에이전트가 분석 시작 전에 70-playbook/을 참조하도록 CLAUDE.md 참조 라우팅에 포함:

```markdown
## 참조 라우팅
| 상황 | 읽을 파일 |
|------|----------|
| 과거 실패 교훈 확인 | → 70-playbook/anti-patterns/ |
| 성공 패턴 참조 | → 70-playbook/patterns/ |
| 시장 관찰 메모 | → 70-playbook/insights/ |
```

이렇게 하면 에이전트가 새 분석 시작 시 과거 교훈을 자동으로 참조한다.

# 에이전트 아키타입 참조 v3

## Chairman 패턴 (모든 도메인 공통)

Chairman은 별도 에이전트가 아닌 **메인 Claude 세션**이 담당한다.

```
역할: 독립 에이전트들의 결과를 수집·종합·최종 판단
담당: 메인 세션
모델: 구축 시 사용자가 결정 (AGENTS.md model_config에 기록)
      → 새 모델 출시 시 model_config만 업데이트
아웃풋: Go / Pivot / No-Go / 추가분석
```

Chairman 판단 형식:
```
━━━━━━━━━━━━━━━━━━━━━━━━
🎖️ Chairman 종합
━━━━━━━━━━━━━━━━━━━━━━━━
판정: [Go / Pivot / No-Go]
신뢰도: [높음 / 중간 / 낮음]
핵심 근거: [1~2문장]
다음 단계: [액션 아이템]
```

---

## 독립성 원칙 (모든 에이전트 공통)

에이전트 파일에 반드시 포함:

```markdown
## 독립성 선언
- 다른 에이전트 결과를 보기 전에 분석한다
- 사용자의 기대 방향에 동조하지 않는다
- 데이터가 말하는 것만 말한다
- 출처 없는 주장은 하지 않는다
```

---

## 모델 선택 가이드

모델명을 하드코딩하지 않는다. 항상 사용자가 결정한다.

**질문 방식:**
```
현재 사용 가능한 Claude 모델을 확인하겠습니다.
[웹 검색: "Anthropic Claude 최신 모델 목록 {현재 연도}"]

확인된 모델 목록:
- [모델A]: [특징 — 추론 깊이, 속도, 비용]
- [모델B]: [특징]
- [모델C]: [특징]

역할별 권장 기준:
┌─────────────┬──────────────────────────────────────┐
│ 역할        │ 권장 모델 특성                          │
├─────────────┼──────────────────────────────────────┤
│ Chairman    │ 추론 깊이 최우선 → 최고 성능 모델       │
│ 독립 에이전트│ 속도·비용 균형 → 중간 성능 모델         │
│ Evaluator   │ 정확성 중요 → 고성능 또는 최고 성능     │
└─────────────┴──────────────────────────────────────┘

어떤 모델로 각 역할을 구성하시겠어요?
```

**AGENTS.md model_config 형식:**
```yaml
model_config:
  chairman: "[사용자 선택 모델명]"
  agents: "[사용자 선택 모델명]"
  evaluator: "[사용자 선택 모델명]"
  decision_date: "YYYY-MM-DD"
  review_trigger: "새 Claude 모델 출시 시 재검토"
```

## 에이전트 수 가이드라인

| 복잡도 | 에이전트 수 | 비고 |
|--------|------------|------|
| 단순 | 2~3개 | 긍정 + 부정 최소 구성 |
| 중간 | 4~5개 | 관점 균형 + Chairman |
| 복잡 | 6~8개 | 세분화, 역할 중복 주의 |
| 8개 초과 | 비권장 | 조율 비용 > 이득 |
